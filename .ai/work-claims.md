# Work Claims

> 防止多个 AI 会话同时修改同一文件或同一功能。
> 每个会话开始工作前必须声明，结束时必须释放。

---

## 使用规则

### 声明工作
```
1. 检查本文件，确认没有冲突的活跃声明
2. 添加声明条目（包含：会话ID、任务描述、涉及的文件、开始时间）
3. 开始工作
```

### 释放工作
```
1. 将声明状态改为 [已完成] 或 [已放弃]
2. 添加结束时间和结果摘要
3. 如果有未完成的部分，在"交接说明"中描述
```

### 发现冲突
```
1. 如果要修改的文件已被其他会话声明 → 等待该声明释放，或协商
2. 如果声明超过 2 小时未更新 → 视为过期，可接手
3. 如果声明标记为 [进行中-阻塞] → 需要人工介入
```

---

## 声明格式

```
### [状态] 任务描述
- 会话: {工具名-会话标识}
- 文件: {涉及的文件路径列表}
- 开始: {YYYY-MM-DD HH:mm}
- 结束: {YYYY-MM-DD HH:mm}（已完成/已放弃时填写）
- 交接: {下一步做什么，或"无"}
```

状态：[进行中] | [进行中-阻塞] | [已完成] | [已放弃]
