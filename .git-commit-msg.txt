fix: path compat, SQL sanitization, providerId priority, scan coverage

1. Rename [beatId] directory to $beatId for Windows PowerShell compatibility
   - Square brackets are wildcard characters in PowerShell, causing path
     resolution failures with Get-Content/Get-ChildItem -Include
   - $beatId is equally supported by React Router v7
   - Update router.tsx import path and api-routes-manager.js reference

2. Sanitize all dynamic SQL table/column names in sync engine
   - conflict-resolution.ts: wrap tableName/pk with sanitizeIdentifier()
   - remote-changes.ts: wrap all dynamic identifiers in SQL statements
   - changelog.ts: wrap tableName/pkColumn in SELECT/UPDATE/DELETE/PRAGMA
   - Also validate column names from remote data with regex before use

3. Add providerId priority matching in api-gateway.ts
   - When providerId is known, use pluginRegistry.selectById() first
   - Fall back to URL/model matching via pluginRegistry.select()
   - Prevents wrong provider selection when multiple providers share URLs

4. Extend check-architecture.mjs SQL scan coverage
   - Scan src/shared/ in addition to src/modules/
