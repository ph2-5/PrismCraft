## 一、产品定位

AI Animation Studio 是一款**桌面端 AI 动画制作工具**，覆盖从故事创作到视频生成的完整工作流：

```
故事构思 → 角色设计 → 场景搭建 → 分镜编排 → AI 视频生成 → 导出成品
```

**核心卖点**：
- **本地优先**：所有数据存储在本地 SQLite 数据库，无需联网即可管理项目
- **多模型支持**：集成 10+ 家 AI 视频/图片/文本提供商，支持用户自制插件扩展
- **完整工作流**：不是单一工具，是覆盖动画制作全流程的集成环境
- **插件生态**：声明式插件系统，用户可自制插件接入新模型

## 二、功能模块一览

### 2.1 故事创作（Story）

| 功能 | 说明 |
|------|------|
| AI 故事规划 | 输入故事梗概，AI 自动生成分镜大纲（支持 Few-Shot 增强） |
| 分镜编辑 | 可视化分镜列表，拖拽排序，支持专业/快速两种编辑模式 |
| 版本管理 | 自动保存故事版本，支持回滚和对比（最多保留 50 个版本） |
| 模板系统 | 内置故事模板，支持导入/导出/分享模板 |
| 元素绑定 | 分镜与角色/场景自动绑定，保持视觉一致性 |

### 2.2 角色管理（Character）

| 功能 | 说明 |
|------|------|
| 角色库 | 创建、编辑、删除角色，支持搜索和筛选 |
| AI 角色图生成 | 根据角色描述自动生成角色形象图 |
| 服装系统 | 为角色配置多套服装，支持换装 |
| 外观属性 | 身高、体型、发型、发色、瞳色等结构化属性 |
| 引用检查 | 删除角色前自动检查是否被故事引用，防止误删 |

### 2.3 场景管理（Scene）

| 功能 | 说明 |
|------|------|
| 场景库 | 与角色模块对称的场景管理系统 |
| AI 场景图生成 | 根据场景描述自动生成场景氛围图 |
| 环境属性 | 天气、时间、情绪、色彩、光线等结构化配置 |
| 场景类型 | 室内/室外/自然/城市/幻想等多种预设类型 |

### 2.4 分镜系统（Shot）

| 功能 | 说明 |
|------|------|
| 分镜生成 | AI 根据故事文本自动生成分镜描述 |
| 镜头参数 | 景别（特写/近景/中景/全景）、机位角度、运动方式 |
| 视觉一致性检查 | 自动检测分镜间的角色/场景一致性，防止穿帮 |
| 特征锚定 | 锁定角色关键特征，确保跨分镜形象统一 |
| 引用系统 | 分镜间链式引用，后一分镜可引用前一帧作为参考 |

### 2.5 视频生成（Video）

| 功能 | 说明 |
|------|------|
| 多提供商支持 | 快手 Kling、MiniMax、PixVerse、Runway、Pika 等 10+ 家 |
| 生成模式 | 关键帧 / 首尾帧 / 直接视频 / 完整工作流 |
| 批量生成 | 支持多个分镜串行/并行批量生成 |
| 任务管理 | 任务状态实时追踪（pending → generating → completed/failed） |
| 智能恢复 | 失败任务自动检测原因，决策重试/跳过/替换策略 |
| 视频缓存 | 本地双层缓存（内存 + IndexedDB），避免重复下载 |

### 2.6 提示词引擎（Prompt）

| 功能 | 说明 |
|------|------|
| 专业模式 | 手动编写完整提示词，适合高级用户 |
| 增强模式 | AI 辅助优化提示词，自动补充质量标签 |
| 快速模式 | 选择预设风格/时长/分辨率，一键生成 |
| 提示词预览 | 实时预览最终发送给 AI 的提示词内容 |
| AI 辅助编辑 | 浮动助手随时优化当前提示词 |

### 2.7 资产管理（Asset）

| 功能 | 说明 |
|------|------|
| 资产库 | 统一管理角色、场景、分镜、媒体文件 |
| 媒体上传 | 支持图片/视频上传，本地存储 |
| 收藏集 | 将常用资产组织为收藏集，快速调用 |
| 导入导出 | 项目数据 JSON 格式备份与恢复 |
| 批量操作 | 批量删除、批量导出 |

### 2.8 数据同步（Sync）

| 功能 | 说明 |
|------|------|
| 多设备同步 | 基于向量时钟的冲突检测与解决 |
| 离线优先 | 无网络时正常工作，联网后自动同步 |
| 冲突策略 | 本地优先 / 远程优先 / 最后写入优先 / 手动选择 |

### 2.9 插件系统（Plugin）

| 功能 | 说明 |
|------|------|
| 内置提供商 | 10+ 家 AI 提供商预置配置 |
| 用户插件 | 通过 JSON 配置接入新模型，无需写代码 |
| 插件市场 | 插件规范文档 + Schema 校验，支持分享插件 |
| 能力映射 | 文本/图片/视频/视觉能力灵活配置和降级 |

## 三、技术架构亮点（买家关心的）

### 3.1 为什么选择这套技术栈

| 技术 | 用途 | 优势 |
|------|------|------|
| Electron | 桌面应用框架 | 跨平台（Win/Mac/Linux），前端技术栈开发 |
| Vite + React | UI 构建与框架 | 极速 HMR、组件化、代码分割优化 |
| React Router | 客户端路由 | SPA 路由管理，懒加载页面级组件 |
| SQLite + better-sqlite3 | 本地数据库 | 零配置、事务安全、WAL 模式高性能 |
| TypeScript（strict） | 开发语言 | 类型安全、重构友好、降低维护成本 |
| Zustand + React Query | 状态管理 | 轻量级客户端状态 + 服务端状态分离 |
| Tailwind CSS | 样式框架 | 原子化 CSS，主题系统，深色/浅色模式 |

### 3.2 代码质量指标

| 指标 | 数值 | 说明 |
|------|------|------|
| 单元测试 | 3920 个 | Vitest 框架，全部通过 |
| E2E 测试 | 290+ 个 | Playwright 浏览器模式 + Electron 模式 |
| 类型检查 | 严格模式 | 生产构建不忽略类型错误，0 error |
| 架构扫描 | 通过 | 自动检查 DDD 依赖方向违规 |
| 代码规范 | ESLint | 0 error |
| 回归守卫 | 76 条 | 7 大类，防止已知 Bug 模式重现 |
| 国际化 | 1634 键 | 全量 i18n 覆盖，用户可见字符串零硬编码 |

### 3.3 安全设计

| 层面 | 措施 |
|------|------|
| API 密钥 | 加密存储（electron-store），不落地明文 |
| 网络请求 | 信任用户配置 URL，仅拦截云元数据端点 |
| 日志脱敏 | 自动脱敏 API Key / Secret / Token |
| SQL 安全 | 参数化查询 + DDL 语句拦截 |
| IPC 权限 | 5 级权限分级，速率限制 |
| 错误恢复 | 崩溃自动恢复、GPU 进程崩溃重载、渲染进程崩溃重建 |

### 3.4 架构设计

| 特性 | 说明 |
|------|------|
| DDD 四层架构 | domain → shared → infrastructure → modules，ESLint 强制依赖方向 |
| DI 容器 | 依赖注入，模块通过 Port 接口解耦基础设施实现 |
| 契约驱动 | MODULE.md + contract.json + index.ts 三级契约，AI 维护友好 |
| JSON 容器模式 | 数据库 JSON 列存储易变字段，避免 ALTER TABLE |
| 声明式 Schema | schema-builder 声明式建表 + 事务性迁移框架 |

| 扩展方向 | 难度 | 说明 |

## 八、已知限制

| 限制 | 说明 | 解决方向 |
|------|------|---------|
| 需自备 AI API Key | 工具本身不提供 AI 服务，需用户自行购买 | 可集成充值/代理模式 |
| 无内置素材库 | 角色/场景图需 AI 生成或用户上传 | 可接入素材市场 |
| 单用户本地使用 | 无多用户协作 | 可扩展服务端版本 |
| 视频生成依赖第三方 | 生成速度和价格受提供商限制 | 插件系统已支持多提供商切换 |
| Windows 安装包未签名 | 会触发 SmartScreen 警告 | 需购买代码签名证书 |
