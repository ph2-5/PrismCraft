## 1. 安装与启动

### 1.1 系统要求

| 操作系统 | 最低版本 | 备注 |
|----------|---------|------|
| Windows | 10 (1903+) | 64 位 |
| macOS | 11 (Big Sur) | Intel + Apple Silicon |
| Linux | Ubuntu 20.04+ | 需要 libgtk-3 |

### 1.2 下载安装

从 [GitHub Releases](https://github.com/ph2-5/PrismCraft/releases) 下载对应平台的安装包：

- **Windows**：`PrismCraft-Setup.exe`（NSIS 安装器，可选安装路径）
- **macOS**：`PrismCraft-x64.dmg`（Intel）或 `PrismCraft-arm64.dmg`（Apple Silicon）
- **Linux**：`PrismCraft.AppImage` 或 `.deb`

> **macOS 用户**：首次打开可能提示"无法验证开发者"。右键点击应用图标 → 选择"打开" → 在弹窗中确认"打开"即可。这是因应用未签名导致的，不影响安全性。

> **Windows 用户**：SmartScreen 可能弹出警告。点击"更多信息" → "仍要运行"即可。

### 1.3 启动应用

安装完成后，从开始菜单（Windows）、Launchpad（macOS）或应用菜单（Linux）启动 PrismCraft。

应用启动后，你会看到左侧导航栏和右侧主内容区。首次启动时，系统信息卡片会显示"待配置"状态。

## 2. 首次配置：API Key

PrismCraft 需要配置至少一个 AI 服务商的 API Key 才能使用生成功能。

### 2.1 进入设置页面

点击左侧导航栏底部的 **设置** 图标，进入设置页面。

### 2.2 配置 API 服务商

在 **API 配置** 标签页中：

1. **选择服务商**：PrismCraft 原生支持 13+ 家 AI 服务商（Kling、Pika、Runway、MiniMax、OpenAI、Anthropic、Google Gemini 等），也支持通过声明式 JSON 插件扩展。

2. **填写信息**：
   - **API URL**：服务商的 API 地址（通常已预填默认值）
   - **API Key**：你在服务商平台申请的密钥
   - **模型名称**：根据服务商文档填写（如 `kling-v2`、`claude-3-5-sonnet-20241022`）

3. **测试连接**：点击"测试连接"按钮，确认配置正确。

4. **保存**：点击保存。API Key 会通过系统级加密存储（Windows Credential Manager / macOS Keychain），不会明文保存。

### 2.3 获取 API Key

| 服务商 | 获取地址 | 用途 |
|--------|---------|------|
| 火山引擎（Kling） | console.volcengine.com | 视频生成 |
| Pika | pika.art | 视频生成 |
| Runway | runwayml.com | 视频生成 |
| MiniMax | platform.minimaxi.com | 视频/图像/文本 |
| OpenAI | platform.openai.com | 文本/图像/视频（Sora） |
| Anthropic | console.anthropic.com | 文本（Claude） |
| Google | ai.google.dev | 文本/图像（Gemini） |

### 2.4 配置建议

- **文本生成**：推荐 OpenAI（GPT-4o）或 Anthropic（Claude 3.5 Sonnet），用于故事规划和 Agent 助手
- **图像生成**：推荐 MiniMax 或 OpenAI（DALL-E），用于角色/场景/关键帧生成
- **视频生成**：推荐 Kling 或 Pika，用于分镜视频生成

> **提示**：你可以同时配置多个服务商，在不同场景下选择最优的模型。Agent 助手会根据任务自动选择合适的模型。

## 3. 核心功能教程

PrismCraft 覆盖完整的动画制作工作流：

```
故事构思 → 角色设计 → 场景搭建 → 分镜编排 → AI 视频生成 → 导出成品
```

### 3.1 角色管理

**路径**：左侧导航 → 角色

1. **创建角色**：点击"新建角色"按钮
2. **填写信息**：
   - 名称、描述、性格特征
   - 可选：上传参考图或通过 AI 生成角色图像
3. **AI 生成图像**：填写角色描述后，点击"AI 生成图像"，选择已配置的图像服务商
4. **搜索**：角色数量多时，使用列表顶部的搜索框（支持名称模糊搜索，200ms 防抖）

> 角色创建后可被故事分镜引用，确保跨分镜视觉一致性。

### 3.2 场景管理

**路径**：左侧导航 → 场景

操作与角色管理类似：
1. 创建场景，填写名称、描述、环境设定
2. AI 生成场景图像或上传参考图
3. 场景可被分镜引用，保持场景视觉统一

### 3.3 故事创作

**路径**：左侧导航 → 故事

故事是 PrismCraft 的核心，采用"节拍（Beat）"系统组织叙事。

1. **创建故事**：点击"新建故事"，填写标题和概述
2. **AI 规划故事**：点击"AI 规划"，输入故事主题，AI 会自动生成节拍列表
3. **编辑节拍**：
   - 每个节拍包含标题、描述、关键帧
   - 可拖拽排序节拍
   - 可为节拍绑定角色/场景/道具
4. **生成关键帧**：选中节拍，点击"生成关键帧"，AI 会根据节拍描述和绑定的元素生成图像
5. **生成视频**：选中节拍，点击"生成视频"，AI 会根据关键帧和描述生成视频片段

### 3.4 分镜编排

在故事详情页，你可以：
- 查看所有节拍的时间线视图
- 为每个节拍生成首尾帧（控制视频起止画面）
- 批量生成关键帧链（连续生成多个节拍的关键帧）
- 检查元素一致性（确保角色/场景在分镜间一致）

### 3.5 视频任务管理

**路径**：左侧导航 → 视频任务

- 查看所有视频生成任务的状态（排队中/生成中/已完成/失败）
- 自动轮询任务状态
- 支持取消、重试、恢复任务
- 已完成的视频可预览和导出

### 3.6 快速生成

**路径**：左侧导航 → 快速生成

不需要创建完整故事，直接输入提示词即可生成视频。适合快速测试 AI 模型效果。

### 3.7 导出

在故事详情页或视频任务页面，可以将生成的视频导出为本地文件。

## 4. AI Agent 助手

**路径**：左侧导航 → Agent（或应用内聊天入口）

PrismCraft 内置 AI Agent 助手，可以帮你完成创作全流程。

### 4.1 Agent 能力

Agent 集成了 141 个工具，覆盖 20 个功能域：

| 域 | 能力示例 |
|----|---------|
| 故事管理 | 创建/编辑/删除故事、生成节拍 |
| 角色管理 | 创建角色、生成图像、分析参考图 |
| 场景管理 | 创建场景、生成图像 |
| 视频生成 | 生成视频、查询状态、恢复任务 |
| 分镜编排 | 生成关键帧、首尾帧、完整工作流 |
| 文件操作 | 读取/写入/管理文件 |
| 配置管理 | 读取/设置 API 配置 |
| 一致性检查 | 角色引用检查、视觉一致性对比 |

### 4.2 使用方式

直接用自然语言描述你的需求，例如：

- "帮我创建一个关于太空探索的故事，生成 5 个节拍"
- "为第一个节拍生成关键帧，使用 Kling 模型"
- "检查所有分镜中角色是否一致"
- "把已完成的视频导出到桌面"

Agent 会自动理解意图、调用工具、返回结果。

### 4.3 注意事项

- Agent 需要**文本模型**支持 function calling（推荐 OpenAI GPT-4o 或 Anthropic Claude 3.5）
- Agent 执行操作前会显示工具调用计划，你可以确认或取消
- Agent 的对话历史保存在当前会话中，关闭后清空

## 5. 设置与系统

### 5.1 API 配置

管理所有 AI 服务商的 API Key 和模型配置。详见 [第 2 节](#2-首次配置api-key)。

### 5.2 自动保存

- **启用自动保存**：开启后，数据变更会自动持久化到本地 SQLite 数据库
- **保存间隔**：默认 30 秒，可调整（10-300 秒）

### 5.3 同步设置

如果你配置了远程同步服务，可以在此设置：
- 同步服务器地址
- 自动同步间隔
- 手动触发同步

### 5.4 系统状态

显示当前系统信息：
- **磁盘空间**：可用/总容量
- **总项目数**：已创建的故事数量
- **版本号**：当前应用版本
- **运行时长**：本次启动后的运行时间
- **检查更新**：手动检查是否有新版本

### 5.5 向量模型（Embedding）

用于 Agent 记忆系统的语义检索。可选择：
- **API 向量模型**：通过已配置的 API 生成向量（推荐）
- **本地向量模型**：上传 ONNX 模型，离线生成向量
- **关键词匹配**：无向量模型时的兜底方案

## 6. 自动更新

PrismCraft 支持自动更新，确保你始终使用最新版本。

### 6.1 自动检查

应用启动时会自动检查更新。如果发现新版本：

1. 顶部弹出通知条："发现新版本 vX.X.X"
2. 应用自动在后台下载更新包
3. 下载完成后通知变为："更新已下载，重启后生效"
4. 点击"重启应用"按钮，应用将关闭并安装更新

### 6.2 手动检查

在 **设置 → 系统状态** 标签页，点击版本号下方的"检查更新"按钮。

### 6.3 注意事项

- 自动更新需要网络连接
- 更新包下载在后台进行，不影响正常使用
- 如果更新失败，通知条会显示错误信息，可稍后重试

## 7. 常见问题 FAQ

### Q1：应用启动后白屏怎么办？

**A**：可能是 better-sqlite3 原生模块加载失败。尝试：
1. 关闭应用
2. 重新安装应用
3. 如果是从源码运行，执行 `npm run rebuild`

### Q2：API Key 保存在哪里？安全吗？

**A**：API Key 通过系统级加密存储：
- **macOS**：Keychain
- **Windows**：Credential Manager
- 不会以明文形式保存到磁盘
- `config/get` API 返回时会自动脱敏（如 `sk-****xxxx`）

### Q3：视频生成失败怎么办？

**A**：
1. 检查 API Key 是否正确（设置 → 测试连接）
2. 检查网络连接
3. 查看视频任务页面的错误信息
4. 部分模型有速率限制，稍后重试
5. 如果任务卡在"生成中"，可尝试"恢复"功能

### Q4：Agent 助手不工作？

**A**：Agent 需要文本模型支持 function calling：
1. 确认已配置支持 function calling 的模型（推荐 OpenAI GPT-4o 或 Anthropic Claude）
2. Anthropic 模型需使用原生 Messages API（v1.2.2+ 已支持）
3. Google Gemini 需使用 OpenAI 兼容端点（v1.2.2+ 已支持）
4. 检查 API Key 是否有足够额度

### Q5：数据存储在哪里？

**A**：所有数据存储在本地：
- **数据库**：`<userData>/ai-animation-studio/data.db`（SQLite）
- **缓存文件**：`<userData>/ai-animation-studio/cache/`
- **日志文件**：`<userData>/ai-animation-studio/logs/`
- **崩溃报告**：`<userData>/ai-animation-studio/Crashes/`

> `userData` 路径：
> - Windows：`C:\Users\<用户名>\AppData\Roaming\`
> - macOS：`~/Library/Application Support/`
> - Linux：`~/.config/`

### Q6：如何备份数据？

**A**：
1. 关闭应用
2. 复制 `<userData>/ai-animation-studio/` 整个目录
3. 恢复时将备份目录覆盖回去

### Q7：支持哪些 AI 服务商？

**A**：原生支持 13+ 家：
- 视频：Kling、Pika、Runway、MiniMax、PixVerse、Seedance、Luma、Sora
- 图像：MiniMax、OpenAI DALL-E
- 文本：OpenAI、Anthropic、Google Gemini、DeepSeek
- 还支持通过声明式 JSON 插件零代码扩展新服务商

### Q8：如何添加自定义 AI 服务商？

**A**：
1. 进入 **设置 → API 配置**
2. 点击"添加插件"
3. 填写插件配置（API URL、请求格式、响应格式等）
4. 或选择模板（openai-compatible / custom-api / image-only）
5. 保存后即可在生成功能中使用

详见 [插件开发规范](plugin-specification.md)。

### Q9：macOS 上提示"应用已损坏"怎么办？

**A**：这是未签名应用的常见问题。在终端执行：
```bash
xattr -cr /Applications/PrismCraft.app
```
然后重新打开应用。

### Q10：如何导出我的作品？

**A**：
1. **单个视频**：在视频任务页面，点击已完成任务的"导出"按钮
2. **故事数据**：在故事详情页，选择"导出数据"（包含故事结构、节拍、绑定关系，不含视频文件）
3. **完整备份**：参考 [Q6](#q6如何备份数据)
